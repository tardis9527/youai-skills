# 产品简报：TeamLog（团队日报助手）

> 使用 Skill：[02_product-discovery](../skills/02_product-discovery.md)
> 日期：2025-04-15
> 本文为精简示例，仅展示产品简报（Product Brief）的最终产出结构

---

## 1. 产品定位

- **一句话描述**：一款面向中小技术团队的 AI 日报助手，自动汇总团队进展，让管理者 30 秒掌握团队状态。
- **目标市场**：10-50 人规模的技术团队 / 研发部门
- **核心价值主张**：告别"日报写了没人看"——AI 自动提炼关键进展、阻塞和风险，替代人工汇总。

---

## 2. 用户角色

### 角色 1：技术团队 Leader（🔴 主要）

- **是谁**：带 5-15 人团队的技术经理/CTO
- **典型画像**：30-40 岁，技术出身转管理，日常会议多，无暇逐条看日报
- **核心任务**：快速了解团队进展、识别阻塞项、向上汇报
- **当前方案**：飞书群里@全员发日报 → 手动逐条阅读 → Excel 汇总
- **最大痛点**：每天花 30-60 分钟读日报和汇总，效率极低
- **成功标准**：每天只花 5 分钟就能掌握团队状态并完成向上汇报
- **使用频率**：每日 1-2 次，每次 5-10 分钟

### 角色 2：开发工程师（🔵 次要）

- **是谁**：一线开发者
- **核心任务**：快速完成日报填写，不想花太多时间
- **当前方案**：飞书/钉钉群里打字发日报，格式随意
- **最大痛点**：每天花 10-15 分钟写日报，觉得浪费时间
- **成功标准**：3 分钟内完成日报，系统自动关联代码提交记录

## 3. MVP 功能清单（一期必做）

| 序号 | 功能 | 用户故事 | 为什么是 MVP |
|------|------|---------|------------|
| 1 | 结构化日报填写 | 作为开发者，我想用模板快速填写日报，以便减少写日报的时间 | 核心数据入口，不做则无数据可汇总 |
| 2 | AI 日报汇总 | 作为 Leader，我想看到 AI 自动生成的团队日报摘要，以便快速了解进展 | 产品核心价值，不做则与飞书群无差异 |
| 3 | 进度看板 | 作为 Leader，我想在看板上看到按人/项目维度的周进度，以便发现进度异常 | 可视化是 Leader 的核心使用场景 |
| 4 | 团队管理 | 作为 Leader，我想创建团队并邀请成员，以便团队成员在同一空间协作 | 多人协作的基础设施 |
| 5 | 通知推送 | 作为 Leader，我想在飞书收到 AI 汇总通知，以便不用打开新工具 | 降低使用门槛，融入现有工作流 |

---

## 4. 核心用户旅程

### 旅程：Leader 查看 AI 日报汇总

角色：技术团队 Leader
目标：快速掌握团队今日进展

步骤：
1. 系统 → 每日 20:00 自动触发 AI 汇总
2. 系统 → 将汇总结果推送到 Leader 的飞书
3. 用户 → 在飞书中点击链接，跳转到 TeamLog 看板
4. 用户 → 浏览 AI 摘要：关键进展（3-5条）、阻塞项（标红）、风险预警
5. 用户 → 点击某条阻塞项，查看原始日报详情
6. 用户 → 在阻塞项下方 @相关人员并留言
7. ✅ 完成：Leader 5 分钟内完成团队状态检视

异常分支：
- 当日日报提交不足 50% → 汇总中标注"数据不完整"，通知未提交成员

## 5. 可行性评估与技术约束

| 功能 | 技术可行性 | 潜在风险 | 建议 |
|------|-----------|---------|------|
| 结构化日报 | ✅ 常规实现 | 无 | React + Ant Design Form |
| AI 汇总 | ✅ 常规实现 | OpenAI API 成本、响应延迟 | GPT-4o-mini 降本；异步生成 + 缓存 |
| 进度看板 | ✅ 常规实现 | 无 | ECharts / AntV |
| 飞书通知 | ✅ 常规实现 | 飞书 API 限流 | Webhook 方式，简单可靠 |
| Git 关联 | ⚠️ 有挑战 | 需用户授权 GitHub/GitLab | 一期可做手动粘贴链接，二期做自动关联 |

## 6. 明确排除项（Not Doing）

- 一期不做周报/月报自动生成（二期考虑）
