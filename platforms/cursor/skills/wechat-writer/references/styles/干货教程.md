# 风格：干货教程

适合：手把手教学、工具使用、方法拆解、SOP 分享。

## 结构特点
- 开头点明"读完你能做到什么"，给读者一个明确预期。
- 主体按步骤推进，每步一个小标题，可独立执行。
- 每步 = 做什么 + 为什么 + 具体怎么做（给例子）。
- 关键处给"避坑提醒"和"对照示例"。
- 结尾给一个可复用的公式或清单。

## 句式节奏
- 指令式、清晰、不绕弯。
- 多用编号、清单、对照表。
- 关键规则加粗突出。

## 类比手法
- 用大白话解释术语，先降门槛再上干货。

## 标志性表达
- "第一步：……"（但避免连续"首先/其次/最后"的死板措辞，用小标题分步）
- "关键词是'具体'。"
- "公式：A → B → C"

## 禁忌
- 忌只讲概念不给可操作步骤。
- 忌步骤模糊（"优化一下"而不说怎么优化）。
- 忌通篇理论、没有例子。
