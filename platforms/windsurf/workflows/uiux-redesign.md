# UI/UX 设计风格重塑

请按照以下步骤系统性地对当前项目进行 UI/UX 设计重塑：

## 步骤

1. **阅读 Skill 定义**：打开并完整阅读 `skills/05_uiux-redesign.md`，理解角色设定、执行步骤和输出结构要求。如果该文件不存在，请从项目 README 中的链接获取，或直接按以下核心流程执行。

2. **产品上下文理解**：采集产品定位、商业模式、核心目标、用户群体画像信息。如果项目中存在「01_project-analysis」报告或「02_product-discovery」的 Product Brief，优先从中提取。否则向用户提问获取。

3. **现状设计审计**：扫描项目代码中的 UI 框架、样式方案、组件库、主题管理等实现。按视觉层级、配色、排版、间距、组件一致性、交互反馈、导航、移动端适配、可访问性、加载性能共 10 个维度评分。

4. **设计策略定义**：基于产品上下文和审计结论，通过设计调性矩阵帮助用户确定方向，制定 3-5 条设计原则，提出 2-3 个设计方向提案供用户选择。

5. **征求确认**：询问用户选定的设计方向，确认后再进入详细规范制定。

6. **设计规范制定**：输出完整的配色系统、排版系统、间距与布局、核心组件规范、交互与动效规范。

7. **实施方案规划**：制定 Design Token 方案、组件改造计划、分阶段实施路线图、风险与降级策略。

8. **保存方案**：将完整方案保存为 `doc/UI_UX_设计重塑方案_{产品名称}_{YYYYMMDD}.md`。
