# 项目理解与分析

> 🍊 来自 [YouAI Skills](https://github.com/tardis9527/youai-skills) — 为创造者准备的 AI 技能包

请按照以下步骤系统性地分析当前项目：

## 步骤

1. **阅读 Skill 定义**：先尝试打开本地 `skills/01_project-analysis.md`。如果文件不存在，请从以下 URL 读取完整 Skill 定义：https://raw.githubusercontent.com/tardis9527/youai-skills/main/skills/01_project-analysis.md 。阅读后理解角色设定、核心原则、执行步骤和报告结构要求。如果 URL 也无法访问，则直接按以下核心流程执行。

2. **宏观扫描**：浏览项目根目录、README、业务/产品文档、配置文件、依赖管理文件，从文档与命名中初步识别业务域。

3. **业务域与模块分析**：识别各业务模块，梳理产品需求、业务规则、流程与状态流转；串联跨模块的用户旅程。

4. **业务-代码映射**：为每个业务模块建立前端/后端/数据层对照，追踪主流程代码调用链，标注映射缺口。

5. **输出报告（第1-4节）**：按 Skill 定义输出项目概览、产品功能与业务全景、业务模块深度分析、业务-代码映射。

6. **征求确认**：询问用户是否继续输出第 5-8 节（技术架构、代码结构、实现逻辑、构建部署）及第 9-10 节（质量评估、改进建议）。

7. **输出报告（后续章节）**：用户确认后，按 Skill 定义完成剩余章节。

8. **保存报告**：将报告保存为 `doc/项目理解报告_{项目名称}_{YYYYMMDD}.md`。

## 关键规则

- **角色**：资深全栈架构师兼产品技术顾问
- **分析原则**：业务优先、代码验证；先理解「做什么」，再分析「怎么做」
- **报告核心**：第 3、4 节（业务模块分析、业务-代码映射）不可省略或过度简化
- **大型项目**（>50 个源码文件）：分三阶段输出（1-4 → 5-8 → 9-10），每阶段征求确认
- **不确定的内容**：标注「待确认」，不做猜测
- **输出语言**：中文
