# 六维量化评分系统

对初稿打分。满分 10 分，**必须 ≥ 6 分**才算通过。低于 6 分自动定位最低维度修改，最多改 3 轮。

## 六个评分维度（各约 1.67 分，合计 10 分）

| 维度 | 看什么 | 高分特征 | 低分特征 |
|---|---|---|---|
| **1. 人味 / 去AI味** | 是否读起来像真人写的 | 有主观判断、有情绪、口语自然 | 通篇套话、空洞、说明书腔 |
| **2. 观点锐度** | 有没有自己的态度和洞见 | 有反直觉判断、敢下结论 | 只罗列信息、不表态 |
| **3. 故事与细节** | 有没有具体场景/数字/转折 | 画面感强、有具体案例 | 全是抽象概括 |
| **4. 结构节奏** | 读起来顺不顺、有没有停顿感 | 长短句搭配、逻辑流过渡 | 大长句堆叠、机械分点 |
| **5. 开头结尾** | 开头抓不抓人、结尾有没有回味 | 悬念/金句开场，结尾有行动号召 | 平铺直叙、虎头蛇尾 |
| **6. 主题契合** | 是否紧扣采访确认的核心观点 | 观点贯穿、不跑题 | 偏离方向、堆砌无关内容 |

每维给 0–1.67 分，也可整体按 10 分制给，只要维度分项理由写清楚。

## 四项一票否决（硬性条件，任一不满足即整体不通过，无论总分多高）

1. **禁止词 ≤ 2 个**（对照 `去AI味规则.md` 板块一统计）
2. **超过 30 字的长句 ≤ 3 个**（按标点断句统计字数）
3. **至少 3 处个人主观表达**（"我"的判断 / 经验 / 吐槽）
4. **不出现连续的"首先、其次、最后"**

## 评分输出格式

```
【六维评分】
1. 人味/去AI味：x.x —— 理由
2. 观点锐度：x.x —— 理由
3. 故事与细节：x.x —— 理由
4. 结构节奏：x.x —— 理由
5. 开头结尾：x.x —— 理由
6. 主题契合：x.x —— 理由
总分：x.x / 10

【一票否决检查】
✔/✘ 禁止词：N 个（限 ≤2）
✔/✘ 超30字长句：N 句（限 ≤3）
✔/✘ 主观表达：N 处（限 ≥3）
✔/✘ 无连续"首先/其次/最后"

结论：通过 / 不通过（第 N 轮自动修改）
```

## 修改循环

不通过 → 找出最低分维度或未过的一票否决项 → 针对性重写 → 重新评分。
最多 3 轮。第 3 轮仍不过 → 标记"需要人工审核"，把具体问题列给用户。
