# 配图设计与生图提示词

> 阶段 8 使用。基于定稿内容，以资深视觉设计师视角设计配图方案，产出可直接粘贴到任意 AI 生图工具（即梦 / Midjourney / Stable Diffusion / GPT-Image 等）使用的中文出图提示词。**本阶段只产出提示词，不调用任何生图接口，不生成真实图片。**
>
> 配套资产：`references/IP配图风格库.md`（6 个固定 IP 风格 + 2 个兜底），第三节从这里推荐风格。

## 一、设计师视角：先选"画哪几张"，再谈"画成什么样"

公众号配图不是给每段都配一张"好看的图"，而是为文章的**核心记忆点**服务。分两层想：

**第 1 层 · 认知锚点选图（画哪几张）**——不要平均配图，通读定稿，只挑最值得可视化的"认知锚点"：
- 核心判断 / 一句话观点、前后对比、输入→输出的闭环、关键转折、常见误区、方法框架的分层。
- 先列一份极简 shot list：每张写清楚**放在哪段后、用途（封面/文中/结尾）、这张图的核心意思、比例**。
- 数量克制：一般封面 1 张 + 文中 1–3 张即可，别把文章做成画册。

**第 2 层 · 风格与画面（画成什么样）**——每张图设计前先问自己：
- 这篇文章最想让读者记住的一个画面/意象是什么？
- 目标读者的审美是什么（专业人群偏克制 / 年轻读者偏活泼 / 情感向偏温暖）？
- 风格从内容和读者推导，**禁止套用"AI 生图默认风格"**（千篇一律的紫蓝渐变、通用赛博朋克、廉价卡通），要让人一眼看出"这是为这篇文章定制的"。

## 二、每张图要确定的信息

| 维度 | 说明 |
|---|---|
| 用途与位置 | 封面图 / 文中插图 / 结尾图；放在正文哪一段之后 |
| 核心意思 | 这张图要让读者记住的**一个**判断/结构/情绪（一张图只讲一个） |
| 画面主体 | 具体是什么（IP 角色 + 物件 / 场景 / 抽象意象） |
| 场景与构图 | 环境、视角（俯视 / 平视 / 特写）、留白位置（封面需为标题预留文字区） |
| 色调与光影 | 主色调、明暗基调（IP 风格自带色彩语义时沿用其规定） |
| 风格关键词 | 3–6 个，不堆砌互相矛盾的风格词 |
| 比例 | 见第六节参考比例 |

## 三、风格方案：从 IP 配图风格库推荐，先给多套供选

**大原则升级**：不再临时罗列通用视觉方向，而是从 `references/IP配图风格库.md` 里，
按本文的**写作风格 / 内容调性 / 目标读者**（用该文件「一、选择指引」的映射表）推荐 **2–4 个候选**，
每个用一两句话说明"为什么适配这篇文章"，让用户选一套（也可要求融合 / 另拟 / 纯纪实摄影）。

- 库里每个 IP 都是一个**固定出场的角色**（墨点 / 暖团 / 方头 / 阿波 / 砚生 / 灯灯），带角色人设、视觉 DNA、适配场景与「角色锚定描述」。
- 两个兜底：**纪实摄影感**（无 IP，硬新闻 / 严肃财经 / 需强真实感时）、**自由融合 / 另拟**。
- 若阶段 0 加载到用户"偏好的配图 IP 风格"，把它放进候选并标注"你上次用的"，但仍给出 1–2 个对照选项，不替用户一锤定音。

> **必须给出至少 2 套供比较，不能只出一套让用户将就。** 选定 IP 后，若该文章要多张图，**全篇统一用同一个 IP**（除非用户要多角色对比）。

### IP 参与核心动作（选了 IP 风格时必守）

选了 IP 风格的图，**IP 必须承担画面的核心动作，不能只是站在角落里的装饰**。自检一句话：
> 如果把 IP 从画面里去掉，这张图的核心意思还完全成立，就说明 IP 太装饰了——重写，让 IP 成为动作主体。

### 原创隐喻生成法（避免每篇长一样、避免照抄案例）

把抽象观点变成一个"看得见的动作"，每篇**重新发明**，不要套用固定套路：
1. **抽象概念 → 物理动作**：卡住 / 漏掉 / 变重 / 分拣 / 沉淀 / 发酵 / 开门 / 折叠 / 拆包 / 回流。
2. **系统结构 → 低科技物件**：坏机器 / 纸箱 / 抽屉 / 水管 / 邮筒 / 秤 / 井 / 梯子 / 怪工位。
3. **让 IP 去做这个动作**：不是站旁边，而是卡在机器里、拉错线、守门、搬运、称重、修补。
> 每次只选 1–2 个物件，不要堆满；动作要服务核心意思，不为怪而怪。

## 四、中文出图提示词公式（适配主流 AI 生图工具）

**选了 IP 风格时（推荐）**——把「角色锚定描述」逐字嵌进去，这是跨图一致的关键：

```
[画面类型] + [IP角色锚定描述（从风格库逐字复制）] + [IP正在做的核心动作与场景] + [该IP的视觉DNA：线条/色彩/背景/质感] + [构图与留白要求] + [少量中文批注词（可选）] + [画面比例] + [负面提示]
```

示例（封面图，犀利点评类文章，选定 IP-4「阿波」波普涂鸦）：

> 一张公众号文章封面图。画面里的固定小角色叫"阿波"：一个粗黑描边、大圆眼、咧嘴、表情夸张、动作浮夸的波普涂鸦小人。阿波举着一块写着"别信"的牌子，瞪大眼睛对着一堆闪着光的"爆款秘诀"翻白眼。高饱和撞色波普涂鸦风格，粗黑描边，明黄正红亮蓝，配速度线和拟声词。横版构图，画面上方三分之一留白供标题文字使用。避免：低饱和、写实精修、性冷淡、温吞、文字乱码、水印。

**选了纪实摄影 / 融合等非 IP 方案时**——用通用公式（无角色锚定块）：

```
[画面类型] + [画面主体与场景] + [构图与视角] + [色调与光影] + [风格关键词] + [画面比例/留白要求] + [负面提示]
```

示例（封面图，严肃财经，纪实摄影兜底）：

> 一张公众号文章封面图，画面主体是清晨交易大厅外一块亮着数字的电子屏，仰视视角，冷调晨光。整体色调克制、偏冷灰蓝，有纪实新闻质感、浅景深。横版构图，上方三分之一留白供标题使用。避免：卡通感、过度饱和、水印、文字乱码。

负面提示统一建议包含：低分辨率、模糊、水印、文字乱码、多余肢体、与整体色调不符的杂色。

## 五、交付前 QA 自查清单

逐张过一遍，任一不满足就调整提示词或换方案：

- [ ] 一张图只讲一个核心意思，没塞成说明书。
- [ ] 选了 IP 的图：IP **承担核心动作**，去掉 IP 后画面就不成立。
- [ ] 隐喻/画面是**为这篇文章现想**的，不是套模板、不像上一篇。
- [ ] 多张图之间角色一致（锚定描述逐字一致、色彩语义统一）。
- [ ] **不是** AI 默认风（紫蓝渐变 / 通用赛博 / 廉价卡通），风格能对上内容和读者。
- [ ] 中文批注少而准（如用），不出现大段文字、不指望模型写对长句。
- [ ] 封面为标题预留了留白区；比例正确。

## 六、参考比例

| 用途 | 建议比例 |
|---|---|
| 方图（头图/小图/卡片图） | 1:1 |
| 横版封面（公众号首图） | ~16:9 |
| 竖版配图（文中竖图/海报，砚生/灯灯尤宜） | 3:4 |

> 具体像素/参数格式因生图工具而异（如 Midjourney 用 `--ar 16:9`，即梦 / Stable Diffusion 各有尺寸选择器），提示词末尾给出比例文字即可，由用户在所用工具里选对应挡位。

## 七、交付方式

- 每张图的提示词以完整、可直接复制粘贴的一段文字呈现（含 IP 锚定块 / 负面提示 / 比例建议）；本阶段只在对话中展示，不主动调用任何生图接口或生成图片文件。
- 逐张展示提示词后，用一句话简述画面重点，方便用户核对；并附一条**一致性操作建议**（同一 seed / 首图当参考图 / 锚定描述别改字，见风格库第四节）。
- 用户想调整 → 换一个 IP 风格重新出提示词，或只微调某张图（换动作/换场景/加减批注），直到满意为止。
- 用户确认满意后，选定的 IP 风格方案与全部提示词会随文字定稿一起，在阶段 9 落盘——单独存为项目 `output/{日期}/{文章标题}/配图提示词.md`（见 SKILL 文档「输出约束」），不需要在本阶段单独保存。
