# 配图设计与生图提示词

> 阶段 8 使用。基于定稿内容，以资深视觉设计师视角设计配图方案，产出可直接粘贴到任意 AI 生图工具（即梦 / Midjourney / Stable Diffusion / GPT-Image 等）使用的中文出图提示词。**本阶段只产出提示词，不调用任何生图接口，不生成真实图片。**

## 一、设计师视角：配图设计方法论

公众号配图不是随便配一张"好看的图"，而是要为文章的核心情绪和记忆点服务。设计前先问自己：

- 这篇文章最想让读者记住的一个画面/意象是什么？
- 用途是什么：封面图（决定分享卡片观感、影响点击）？文中插图（辅助理解、调节阅读节奏）？
- 目标读者的审美偏好是什么（专业人群偏克制 / 年轻读者偏活泼 / 情感向偏温暖）？
- 风格从内容本身推导，**禁止套用"AI 生图默认风格"**（如千篇一律的紫蓝渐变、通用赛博朋克、廉价卡通感），风格关键词要让人一眼看出"这是为这篇文章定制的"，而不是随便一张网图。

## 二、每张图要确定的信息

| 维度 | 说明 |
|---|---|
| 用途 | 封面图 / 文中插图 / 结尾图 |
| 画面主体 | 具体是什么（人物 / 物品 / 场景 / 抽象意象） |
| 场景与构图 | 环境、视角（俯视 / 平视 / 特写）、留白位置（封面图需为标题预留文字区） |
| 色调与光影 | 主色调、明暗基调 |
| 风格关键词 | 3–6 个，不堆砌互相矛盾的风格词 |
| 比例 | 见第五节参考比例 |

## 三、风格方案：先给 2–4 套供用户选择

单凭 AI 一次拍板容易文不对题——先按内容调性推导出 2–4 套不同的视觉风格方向，各自说明关键词与适用理由，让用户挑一套（或要求融合/另拟），再进入第四节按选定风格出提示词。

常见风格方向示例（按内容调性挑，不要机械匹配，也不必局限于以下四种）：

| 方案 | 风格方向 | 关键词 | 适合内容 |
|---|---|---|---|
| A | 编辑摄影感 | 真实光影、纪实构图、浅景深 | 故事叙事、深度评论类文章 |
| B | 扁平插画 / 信息图 | 简洁色块、图形化符号、层次清晰 | 干货教程、方法论类文章 |
| C | 极简线条 / 留白 | 单色线条、大量留白、克制 | 认知深度、观点类文章 |
| D | 复古胶片 / 暖色调 | 暖黄偏色、颗粒感、怀旧光影 | 怀旧、个人化叙事 |

> **必须给出至少 2 套供用户比较，不能只出一套让用户将就。** 展示时用一两句话说明每套方案"为什么适合这篇文章"，而不是简单罗列关键词。

## 四、中文出图提示词公式（通用，适配主流 AI 生图工具）

```
[画面类型] + [画面主体与场景] + [构图与视角] + [色调与光影] + [风格关键词] + [画面比例/留白要求] + [负面提示]
```

示例（封面图，故事叙事类文章，选定方案 A · 编辑摄影感）：

> 一张公众号文章封面图，画面主体是深夜写字楼里一扇亮着灯的窗，俯视城市街道的视角，远处霓虹微光，近处窗台放着一杯已经凉掉的咖啡。整体色调是深蓝与暖黄的对比，光影克制、有电影感。风格：纪实摄影质感、浅景深、轻微颗粒感。横版构图，画面上方三分之一留白供标题文字使用。避免：卡通感、过度饱和、水印、人物正脸特写、文字乱码。

负面提示统一建议包含：低分辨率、模糊、水印、文字乱码、多余肢体、与整体色调不符的杂色。

## 五、参考比例

| 用途 | 建议比例 |
|---|---|
| 方图（头图/小图/卡片图） | 1:1 |
| 横版封面（公众号首图） | ~16:9 |
| 竖版配图（文中竖图/海报） | 3:4 |

> 具体像素/参数格式因生图工具而异（如 Midjourney 用 `--ar 16:9`，即梦 / Stable Diffusion 各有自己的尺寸选择器），提示词末尾给出比例文字即可，由用户在所用工具里选择对应挡位。

## 六、交付方式

- 每张图的提示词以完整、可直接复制粘贴的一段文字呈现（含负面提示、比例建议）；本阶段只在对话中展示，不主动调用任何生图接口或生成图片文件。
- 逐张展示提示词后，用一句话简述这张图的画面重点，方便用户核对是否符合预期。
- 用户想调整 → 换一套风格方案重新出提示词，或只微调某张图的细节（换主体/换色调/加减风格词），直到满意为止。
- 用户确认满意后，选定的风格方案与全部提示词会随文字定稿一起，在阶段 9 落盘——单独存为项目 `output/{日期}/{文章标题}/配图提示词.md`（见 SKILL 文档「输出约束」），不需要在本阶段单独保存。
