# 公众号文章自进化写作

> 🍊 来自 [YouAI Skills](https://github.com/tardis9527/youai-skills) — 为创造者准备的 AI 技能包

一套会自我进化的公众号写作系统：带着记忆开工，采访挖料，写完自审，定稿后以资深设计师视角设计配图并产出多套风格方案的中文出图提示词，再把经验沉淀回知识库。请按以下步骤执行：

## 步骤

1. **阅读 Skill 定义**：先尝试打开本地 `skills/08_wechat-writer.md`。如果文件不存在，请从以下 URL 读取完整 Skill 定义：https://raw.githubusercontent.com/tardis9527/youai-skills/main/skills/08_wechat-writer.md 。阅读后理解角色设定、核心原则与九阶段闭环流程。如果 URL 也无法访问，则直接按以下核心流程执行。

2. **阶段 0 · 学习加载**（静默）：读取本地 `knowledge/` 下的 user-preferences、successful-patterns、common-revisions、execution-history，把用户画像装进脑子，一句话确认后开工。文件为空则视为首次使用。

3. **阶段 1 · 全网调研**（静默，不汇报）：用 WebSearch / WebFetch 搞清主题"是什么 / 为什么值得写 / 大家怎么评价"，带着背景直接进入采访。

4. **阶段 2 · 四轮渐进式采访**：一次只问 1–3 个问题，逐轮深入（破冰定位→深挖具象→灵魂拷问→验证补充），回答笼统就动态追问。

5. **阶段 3 · 整理确认**（停下等用户）：用表格汇总核心观点 / 目标读者 / 关键故事 / 关键数据 / 一句话记忆点，请用户确认方向。

6. **阶段 4 · 风格与参数选择**（停下等用户）：从 `references/styles/` 五种风格选一种，给 3–5 个候选标题，定篇幅与配图方案。

7. **阶段 5 · 撰写初稿**：严格按选定风格文件写，复用成功模式，边写边避开 `references/去AI味规则.md` 的禁止词。

8. **阶段 6 · 六维评分审核**（AI 自审）：先按去 AI 味规则清洗，再按 `references/六维评分.md` 打分，须 ≥ 6 分且过四项一票否决，最多自动改 3 轮。

9. **阶段 7 · 用户把关**（停下等用户）：交付定稿，按用户意见修改到满意。

10. **阶段 8 · 配图设计与出图提示词**（停下等用户）：切换为资深视觉设计师视角，基于定稿推导 2–4 套不同视觉风格方案供用户选择，再按选定风格按 `references/配图设计与生图提示词.md` 的提示词公式产出中文出图提示词，交由用户自行粘贴到任意 AI 生图工具生成；本阶段不调用任何生图接口。

11. **阶段 9 · 落盘与学习记录**（必做）：先在项目 `output/` 目录下按「日期 / 文章标题」建两层子目录 `output/{YYYY-MM-DD}/{文章标题}/`，把产出**拆成素材文件**分别落盘——`正文.md`（文字定稿含标题，必存）、`配图提示词.md`（若做了配图：选定风格方案 + 完整出图提示词，含负面提示；跳过配图则不生成）；标题层去掉文件系统非法字符。再问满意度、追加执行历史（含配图情况）、提取成功模式 / 常见修改 / 新偏好写回 `knowledge/`；每 5 次执行触发一次偏好确认。

## 关键规则

- **带着记忆开始，越用越聪明**：每次先加载 `knowledge/`，定稿后必写回学习点，不记录 = 下次重犯错。
- **定稿必须落盘**：文字定稿与配图提示词最终要**拆成 `正文.md`、`配图提示词.md` 等素材文件**，写入项目 `output/{日期}/{文章标题}/` 目录，不能只留在对话里。
- **人是决策者**：AI 自动推进，只在阶段 3、4、7、8 停下来等用户拍板。
- **规则要具体**：质量判断落到硬指标——禁止词 ≤ 2、超 30 字长句 ≤ 3、≥ 3 处"我"的主观表达、无连续"首先 / 其次 / 最后"。
- **自己过不了的稿子不交付**：先过六维评分再给用户。
- **输出语言**：中文。
